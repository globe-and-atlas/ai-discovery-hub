# Directive: generate_curriculum

## Validation Contract

- [ ] The output satisfies the directive's acceptance criteria or documented success standard.
- [ ] Any deterministic error encountered during execution is recorded in `knowledge/ERRORS.md`.

**Goal**: Expand a short "Lab" exercise from the AI Discovery Hub into a comprehensive, step-by-step learning curriculum using an LLM.

**Inputs**:
- `data/hub-output.json` (generated by `execution/build_hub.py`). This contains the array of generated exercises and their corresponding artifacts.
- Target item number (e.g., `--item 1`).

**Execution Script**:
- `python3 execution/expand_lab.py`

**Usage Options**:
- `python3 execution/expand_lab.py --list` (Lists available exercises from the latest dashboard build)
- `python3 execution/expand_lab.py --item N` (Generates the curriculum for exercise N)

**Expected Output**:
- A detailed Markdown file (`.md`) saved to the `curriculums/` directory containing step-by-step instructions, code snippets, and context.

**Edge Cases**:
- If `hub-output.json` does not exist, the script will abort and request the user to run `build_hub.py` first.
- If the item number is out of bounds, the script will display an error and exit.
- If there is a network error connecting to the LLM API, the script will throw an exception (this mirrors `build_hub.py` behavior).
